Further Study

Let’s add some more features to our calculator.

Pass Inputs As a List

Make a new arithmetic.py with functions that take in a list containing any number of number inputs.

Check User Inputs

Try to break your calculator (or ask one of us to do it for you). Once you find some ways to break it, prevent the user from breaking it in those ways.

Here are some examples:

  • Typing in a word instead of a number

  • Typing in a command incorrectly

  • Typing in too few numbers for a given command

Infinite Inputs!

Re-write calculator.py so that it will work on any number of inputs. Finish reading this section before you get started — it will introduce a new function that will help you accomplish this goal.

Add this line to the top of calculator.py

from functools import reduce

Read about reduce in the functools documentation, here.

Use reduce to re-write calculator.py so that it will work on any number of inputs, using the original arithmetic.py provided.

Rewrite reduce()

Replace the reduce() function you just used with your own, called my_reduce().

Fixed Decimal Places

Set the number of decimal places you would like your results to show.

Automate the Input/Output Process

Have your calculator.py read in the operations (pow 3 2, for instance) from a file rather than from the user.

  • First, print the results when the file is run.

  • Now, try writing the results out to a file. Make a new file for the results and write your results to that file.

Practice Good Style

How can your code be improved? Is it readable? Do you have comments? Could your variable names be improved?

Try a Different Notation

Convert your expression syntax from prefix notation to infix notation. Infix notation is what you’re used to seeing in calculations, and it looks like this:

1 + 2
5 - 6
4 * 3

Ignore the usual operator precedence rules, and just evaluate left to right.
